Module moveCommands

source code

The moveCommands.py file contains all of the methods and functions that are used to control the zaber stages that make up the x-TRAP device.

Note: This set of functions requires the stages to be in the configuration Z, Rotation, X, Y

Author: David Dellsperger

stop(ser, s1, s2, s3, s4, zeroPosition, gui)

source code 

Sends the command to all stages to stop movement, should only be used as an emergency stop function as it causes other problems with the stages after a stop command is sent

Parameters:
  • ser - The Serial port that has the stages connected to and open
  • s1 - String var for stage 1's movement (z-axis)
  • s2 - String var for stage 2's movement (rotary axis)
  • s3 - String var for stage 3's movement (x-axis)
  • s4 - String var for stage 4's movement (y-axis)
  • zeroPosition - 4-tuple containing the set zero position of the stages
  • gui - the reference to the main gui

moveAllRelative(ser, mag1, mag2, mag3, mag4, s1, s2, s3, s4, zeroPosition, currentPosition, gui)

moveOne(ser, dev, mag, cmd)

source code 

This is the call to move one stage based on the magnitude given and command type (21 or 20 for relative/absolute) The magnitude is converted to the number of micro steps and then the send command is used

Parameters:
  • ser - The Serial port that has the stages connected to and open
  • dev - The device number being moved (1-4)
  • mag - The magnitude to move mm for dev 1, 3, 4, and degrees for 2
  • cmd - The command being issued 20 for Absolute move, 21 for Relative move.

setZeroPoint(ser, s1, s2, s3, s4, zeroPosition, gui)

source code 

Initialization step which is required before any other movement command. This homes the devices (in order to reset the zero position of the stage's memory) then moves the stages to the zero position.

Parameters:
  • ser - The Serial port that has the stages connected to and open
  • s1 - String var for stage 1's movement (z-axis)
  • s2 - String var for stage 2's movement (rotary axis)
  • s3 - String var for stage 3's movement (x-axis)
  • s4 - String var for stage 4's movement (y-axis)
  • zeroPosition - 4-tuple containing the set zero position of the stages
  • gui - the reference to the main gui
